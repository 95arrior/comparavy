# Guide Type Standards

These standards define what Comparavy must generate, rewrite, and approve for each guide type.

## How-To

Purpose: solve one concrete problem with a practical AI-supported workflow.

Reader intent: "Show me how to get this done without wasting time."

Required structure:

- Quick Answer
- What you'll make
- One generated prompt builder
- More details toggle for optional fields when useful
- Copy Prompt action
- Check before using checklist
- Real-World Scenario
- What You Need
- Step-by-Step Workflow
- Mobile Workflow
- Desktop Workflow
- Tools You Can Use
- Example Result
- Common Mistakes
- FAQ
- Next Step

First 100 words must include the problem, starting input, desired output, first action, and review warning.
The prompt builder, Copy Prompt, Example output, and Check before using
checklist must appear before long article content. Workflow, tools, and FAQ
belong lower on the page.

Strong article:

- Starts with the answer.
- Shows what the reader has, what they will get, and what details to prepare.
- Gives one generated prompt that starts with the actual task and includes clear placeholders when fields are blank.
- Uses topic-specific top summary, input fields, generated prompt rules,
  example output, checklist, and tone.
- Explains that filling in details improves the result but is not required.
- Shows a real workflow before tool recommendations.
- Makes tools support steps.
- Gives mobile and desktop advice that is not copied.
- Includes an example result.

Weak article:

- Starts with "AI can help."
- Reads like a top tools list.
- Gives generic steps that could apply to any topic.
- Uses shallow device sections.
- Repeats generic boilerplate from another shortcut, such as "Build and copy
  the prompt first," "Use this shortcut when you want the prompt first," "Read
  this after you have the prompt," or "Do the work in order."
- Uses a long multi-sentence `/shortcuts` card summary instead of one short
  value proposition sentence.

Common failures:

- vague quick answer
- too much tool-list content
- generic FAQ
- tool recommendations not integrated into the workflow

Good title: How to Turn Meeting Notes Into a Client Recap with AI

Bad title: Best AI Tools for Better Meeting Productivity

CTA logic: send readers to /finder only after the workflow and tool roles are clear.

## Tool-Decision

Purpose: help the reader choose between tools for one clear job.

Reader intent: "Which tool should I use first, and when should I choose something else?"

Required structure:

- Quick Verdict
- One generated prompt builder when prompts are included
- More details toggle for optional fields when useful
- Check before using checklist when prompt output needs review
- Which One Should You Choose?
- Decision Path
- Best Picks by Situation
- Comparison Criteria
- Comparison Table
- Tool Cards
- Who Should Use This
- Who Should Avoid This
- High-Intent FAQ
- Final Verdict
- Finder CTA

First 100 words must name the best starting tool, the best alternative, when to avoid the top pick, and the main comparison criteria.

Strong article:

- Gives a fast choice.
- Makes any prompt easy to generate, copy, paste, and check without knowing prompt-writing terms.
- Avoids multiple public prompt choices unless the topic truly requires them.
- Uses real decision criteria.
- Explains better/worse fit by situation.
- Uses distinct Best for / Avoid if language.
- Does not make every tool sound equally good.

Weak article:

- Repeats the same Best for / Avoid if text.
- Ranks tools without criteria.
- Hides the recommendation below generic content.
- Uses broad claims like "strong option."

Common failures:

- weak decision path
- repeated tool-card phrasing
- unsupported rankings
- vague final verdict

Good title: Best AI Tools for Turning Meeting Notes Into Client Recaps

Bad title: Top AI Tools Everyone Should Try

CTA logic: use /finder for readers whose workflow, budget, source type, or device needs differ from the shortlist.

## Income

Purpose: explain a realistic AI-supported service or freelance workflow.

Reader intent: "Can I use AI to offer this service, and what would the first realistic step be?"

Required structure:

- Reality Check
- What You Can Offer
- Skill Needed
- Step-by-Step Workflow
- Tools That Help
- First Realistic Step
- Time / Cost / Difficulty
- Limitations
- Mistakes to Avoid
- FAQ
- Final Recommendation

First 100 words must state the realistic offer, skill requirement, effort, and warning that income is not guaranteed.

Strong article:

- Sets expectations.
- Describes a real service package.
- Explains skill, effort, cost, limitations, and first step.
- Avoids income promises.

Weak article:

- Implies easy money.
- Treats AI as a substitute for skill.
- Skips delivery quality.
- Suggests pricing or demand without evidence.

Common failures:

- guaranteed income language
- no first realistic step
- no limitations
- weak skill explanation

Good title: How to Sell AI-Assisted Podcast Clip Packages as a Freelancer

Bad title: Make Money Fast With AI Video Tools

CTA logic: point readers to /finder only after explaining the service workflow and the tool role.

## Trend-Led

Purpose: turn a current AI trend into a practical choice.

Reader intent: "This trend is everywhere. What should I actually use or avoid?"

Required structure:

- Quick Decision
- Why People Are Comparing This
- What Changed or Why It Matters
- Best Picks by Situation
- Comparison Criteria
- Comparison Table
- What to Avoid
- Recommended Workflow
- FAQ
- Final Recommendation

First 100 words must explain the practical choice, who it affects, and what the reader should do now.

Strong article:

- Converts trend interest into a practical decision.
- Avoids unsupported breaking-news claims.
- Explains what changed only when supported.
- Gives a workflow or decision path.

Weak article:

- Pretends to be news without sources.
- Uses trend hype.
- Lists tools without explaining the reader's decision.

Common failures:

- unsupported current claims
- no practical choice
- topic drift
- generic comparison

Good title: Best AI Tools for Source-Grounded Research After NotebookLM Growth

Bad title: The Latest AI Trend You Need Today

CTA logic: use /finder when the reader needs a recommendation based on source type, device, budget, or review depth.
